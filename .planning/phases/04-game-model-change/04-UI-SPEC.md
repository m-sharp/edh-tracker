## Spacing Scale

MUI's default `theme.spacing` unit is 8px. All spacing values expressed as multiples of `theme.spacing(N)`.

| Token | Value | MUI spacing() call | Usage |
|-------|-------|--------------------|-------|
| xs | 4px | `theme.spacing(0.5)` | Icon gaps, inline padding within chips |
| sm | 8px | `theme.spacing(1)` | Compact element spacing, gap between paired buttons |
| md | 16px | `theme.spacing(2)` | Default element spacing, form field separation |
| lg | 24px | `theme.spacing(3)` | Section padding, vertical rhythm between content blocks |
| xl | 32px | `theme.spacing(4)` | Layout gaps between major sections |
| 2xl | 48px | `theme.spacing(6)` | Major section breaks |
| 3xl | 64px | `theme.spacing(8)` | Page-level spacing |

Exceptions:
- Touch targets on mobile: minimum 44px height for all interactive elements (buttons, list items). MUI `size="medium"` satisfies this; `size="small"` must have `sx={{ minHeight: 44 }}` when rendered in touch contexts.
- Player entry card remove button (TooltipIconButton): minimum 44px touch target even though it is icon-only.
- Card internal padding: `theme.spacing(2)` (16px) on all four sides.

**Source:** Inherited from Phase 2 UI-SPEC.md. No spacing token changes. CONTEXT.md D-03 drives card padding decision.

## Typography

All heading roles use Josefin Sans. Body and label roles use Roboto (MUI default — do not override).

| Role | Size | Weight | Line Height | MUI variant | Usage |
|------|------|--------|-------------|-------------|-------|
| Display | 28px | 700 (bold) | 1.2 | h4 | Page-level headings (e.g., "Add New Game") |
| Heading | 20px | 700 (bold) | 1.2 | h6 | Section headings |
| Body | 16px | 400 (regular) | 1.5 | body1 | Primary readable content, form labels |
| Label | 14px | 400 (regular) | 1.4 | body2 | Helper text, secondary info, card sub-labels |

Two weights only: 400 (regular) and 700 (bold). No intermediate weight.

## Color

Dark theme throughout. All values apply to MUI `palette` configuration in `createTheme({ palette: { mode: 'dark', ... } })`.

| Role | Value | MUI palette key | Usage |
|------|-------|-----------------|-------|
| Dominant (60%) | `#0f1117` | `palette.background.default` | App background, body fill, new game form background |
| Secondary (30%) | `#1a1a2e` | `palette.background.paper` | Player entry cards (MUI Card/Paper elevation 1), AppBar |
| Accent (10%) | `#c9a227` | `palette.primary.main` | See accent reserved list below |
| Accent hover | `#e0b83a` | `palette.primary.light` | Hover state for accent elements |
| Primary text | `#e8edf5` | `palette.text.primary` | All primary readable text |
| Secondary text | MUI dark-mode default opacity on `#e8edf5` | `palette.text.secondary` | Helper text, card sub-labels |
| Destructive | MUI `error.main` (`#f44336`) | `palette.error.main` | Remove entry card button (❌) |

**Accent reserved for:**
1. Primary `variant="contained"` buttons — specifically "Submit" (form submit) and "Add Deck" (add entry card)
2. Active Tab indicator underline
3. Focused TextField outline
4. Chip `color="primary"` for role/status indicators that need emphasis

Accent is NOT used for: the remove card icon button (uses error color), secondary/outlined buttons, AppBar text, DataGrid header backgrounds.

**Source:** Inherited from Phase 2 UI-SPEC.md. Remove button uses `color="error"` per CONTEXT.md D-08.

## Component Inventory

New and modified components for this phase:

### NewGameView — full redesign (`app/src/routes/new/index.tsx`)

**Page layout:**
- Outer: `Box sx={{ display: "flex", flexDirection: "column", alignItems: "center", width: "100%", px: 2, pb: 4 }}` — full-width with horizontal padding at small viewports.
- Page heading: `Typography variant="h4"` — "Add New Game" — using Display role (28px, weight 700, Josefin Sans).
- Form container: `Box sx={{ display: "flex", flexDirection: "column", gap: 2, width: "100%", maxWidth: 600 }}` — constrains form to readable width on desktop.

**Format field (always visible, required):**
- `Select` component with `fullWidth`, `required`, `label="Format"`.
- Rendered above the player card list.
- No collapse — always visible per CONTEXT.md D-05.
- When no format is selected, display the placeholder `"Select a format"` (disabled empty MenuItem).

**Description field (collapsed by default):**
- Hidden behind an expand link: `Button variant="text" size="small"` labelled `"+ Add description"`.
- When expanded: `TextField multiline fullWidth label="Description" placeholder="Add a game description!"`.
- Collapse/expand toggles via `useState<boolean>` — no animation required.
- Per CONTEXT.md D-06.

**Player entry cards — stacked list:**
- Each card: `Card variant="outlined" sx={{ width: "100%", position: "relative", p: 2 }}`.
- Use `variant="outlined"` (border, no shadow) over `elevation` — cleaner on dark background.
- Internal layout: `Box sx={{ display: "flex", flexDirection: "column", gap: 1.5 }}` (12px gap between rows).

  **Card header row:**
  - `Box sx={{ display: "flex", justifyContent: "space-between", alignItems: "center", mb: 1 }}`.
  - Left: `Typography variant="body2"` — "Deck {N}" label (e.g., "Deck 1", "Deck 2").
  - Right: `TooltipIconButton` wrapping `CloseIcon` — tooltip `"Remove"`, `color="error"`, `size="small"`, `sx={{ minHeight: 44, minWidth: 44 }}`. Disabled when only 2 cards remain (minimum enforced).

  **Deck Autocomplete (full width):**
  - `Autocomplete` with `fullWidth`, `options={filteredDecks}`.
  - `getOptionLabel={(deck) => \`${deck.name} (${deck.player_name})\`}` — format: `"DeckName (PlayerName)"`. Per CONTEXT.md D-01.
  - `renderInput={(params) => <TextField {...params} label="Deck" required />}`.

  **Place + Kills row:**
  - `Box sx={{ display: "flex", gap: 2 }}`.
  - Place: `TextField type="number" label="Place" required inputProps={{ min: 1 }} sx={{ flex: 1 }}`.
  - Kills: `TextField type="number" label="Kills" required inputProps={{ min: 0 }} sx={{ flex: 1 }}`.
  - Both use `TextField` with `type="number"` — per CONTEXT.md discretion (number inputs over Select for place/kills; simpler for arbitrary player counts).

**Add Deck button:**
- `Button variant="outlined" startIcon={<AddIcon />} sx={{ alignSelf: "flex-start", mt: 1, minHeight: 44 }}` — "Add Deck".
- Per CONTEXT.md D-07 (rename from "Add Player" to "Add Deck").

**Submit button:**
- `Button variant="contained" size="large" startIcon={<PublishIcon />} sx={{ mt: 2, minHeight: 48 }}` — "Submit Game".
- Full width on mobile (`fullWidth` on `xs`, `false` on `sm+` via responsive prop) — or simply `fullWidth` always for simplicity within `maxWidth: 600` container.

**Initial state:**
- 2 player entry cards rendered on load. Per CONTEXT.md D-08 (minimum 2) and discretion (2 as natural default).

**Minimum validation:**
- Remove button disabled (not hidden) when exactly 2 cards remain. Tooltip text when disabled: `"Minimum 2 entries required"`.
- Submit button disabled until: format is selected AND all deck Autocompletes have a value AND all place + kills fields are non-zero.

### GameView — AddResultModal player picker removal (`app/src/routes/game/index.tsx`)

- Remove the `Select` or `Autocomplete` for player from `AddResultModal`.
- Player context is implicitly established via deck ownership — no UI element needed.
- Deck picker label in AddResultModal does NOT change to `"DeckName (PlayerName)"` format — per CONTEXT.md D-02 (that format applies only to NewGameView).
- No other layout changes to AddResultModal; preserve existing structure.

### Record component — dynamic display (`app/src/components/stats.tsx`)

**Current behavior:** Always shows 4 places — `first / second / third / fourth`.

**New behavior:** Iterate from place 1 to `max(keys in record dict)`, show counts separated by ` / `. Places with no entry show `0`.

Examples per CONTEXT.md D-09:
- 4-player game: `5 / 3 / 2 / 1`
- 3-player game: `5 / 3 / 2`
- Mixed (up to place 6): `5 / 3 / 2 / 1 / 0 / 1`

**Implementation pattern:**
```
const maxPlace = Math.max(...Object.keys(record).map(Number), 1);
const parts = Array.from({ length: maxPlace }, (_, i) => record[i + 1] ?? 0);
return <span className="record">{parts.join(" / ")}</span>;
```

**RecordComparator:** Iterate from place 1 up to `Math.max` of both record dicts' keys; return the first non-zero difference. Per CONTEXT.md D-10.

## Visual Hierarchy

**New Game form focal point sequence:**
1. "Add New Game" heading (Display/h4, Josefin Sans, 28px) — establishes context.
2. Format selector — required field, anchors the form before the card list.
3. Player entry cards — primary work area; stacked vertically, full-width on mobile.
4. "Add Deck" + "Submit Game" buttons — action zone at the bottom.

The description field (`"+ Add description"`) is de-emphasized below the format select; its collapsed default removes it from the visual hierarchy until explicitly expanded.
