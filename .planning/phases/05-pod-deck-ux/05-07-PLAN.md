<objective>
Fix the commander creation round-trip: backend returns the new ID, frontend unwraps it.

Purpose: POST /api/commander currently discards the returned ID and returns an empty 201. The
frontend callers in deck/new and SettingsTab call res.json() on the empty body, which throws a
SyntaxError and shows "Failed to create commander. Try again." — even though the commander was
actually created. UAT test 15 (and test 17 by extension).

Output: CommanderCreate writes {"id": N} in the response body. PostCommander in http.ts returns
Promise<{id: number}> matching PostDeck/PostPod style. Caller sites in deck/new and SettingsTab
already do `const { id } = await PostCommander(...)` — they work correctly once PostCommander
is fixed, so they need no changes.

```typescript
export async function PostDeck(body: NewDeckRequest): Promise<{ id: number }> {
    const res = await fetch(`${API_BASE_URL}/api/deck`, {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        credentials: "include",
        body: JSON.stringify(body),
    });
    if (!res.ok) {
        throw new Error(`Request failed: ${res.status}`);
    }
    return await res.json();
}
```
