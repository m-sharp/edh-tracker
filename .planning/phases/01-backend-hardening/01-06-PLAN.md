<objective>
Fix frontend white-screen crash when backend returns non-2xx responses (e.g., 400 validation errors).

Purpose: UAT test 7 revealed that the frontend crashes with `Uncaught SyntaxError: Unexpected token '<'` when the backend returns a 400 for string length validation. The root cause is twofold: (1) functions like PostPod/PostPodJoin/PostPodInvite call `.json()` without checking `res.ok`, and (2) fire-and-forget mutations (PatchPlayer, PatchDeck, PatchPod, DeleteDeck, etc.) silently discard the Response, making catch blocks in call sites dead code.

Output: Updated http.ts with res.ok guards and throwing mutations; updated route files with proper error handling.
</objective>

<task type="auto">
  <name>Task 1: Add res.ok guards and throw on non-2xx in http.ts</name>
  <files>app/src/http.ts</files>
  <action>
Add res.ok guards to all functions in http.ts that either (a) call res.json() without checking res.ok first, or (b) discard the Response entirely (fire-and-forget pattern).

Specifically:

1. **Functions that call res.json() without res.ok guard** — add a guard that throws before the .json() call:
   - `GetPlayer`, `GetPlayers`, `GetPlayersForPod`, `GetDecksForPlayer`, `GetGamesForPlayer` (GET functions)
   - `GetDeck`, `GetDecks`, `GetDecksForPod`, `GetGamesForDeck` (GET functions)
   - `GetGame`, `GetGames`, `GetGamesForPod` (GET functions)
   - `GetFormats`, `GetCommanders`, `GetCommander` (GET functions)
   - `GetPod`, `GetPodsForPlayer`, `GetAllDecksForPod` (GET functions)
   - `PostPod` — POST that returns JSON
   - `PostPodInvite` — POST that returns JSON
   - `PostPodJoin` — POST that returns JSON

   Pattern for these:
   ```typescript
   if (!res.ok) {
     throw new Error(`Request failed: ${res.status}`);
   }
   return res.json();
   ```

2. **Fire-and-forget mutation functions** — currently use `await fetch(...)` with no status check. Add res.ok guard that throws:
   - `PatchPlayer`
   - `PatchDeck`
   - `DeleteDeck`
   - `PatchGame`
   - `DeleteGame`
   - `PostGameResult`
   - `PatchGameResult`
   - `DeleteGameResult`
   - `PatchPod`
   - `DeletePod`
   - `PatchPodPlayerRole`
   - `DeletePodPlayer`

   Pattern for these — capture the response and check:
   ```typescript
   const res = await fetch(...);
   if (!res.ok) {
     throw new Error(`Request failed: ${res.status}`);
   }
   ```

3. **Functions that already return raw Response** — `PostGame` and `PostCommander` return `Promise<Response>`. Leave these unchanged — their callers already handle the response directly.

4. **PostPodLeave** already has a res.ok guard. Leave it unchanged.

Do NOT change function signatures, return types, or the backend error format. Do NOT add toast/snackbar infrastructure. Keep the error messages simple — just include the status code.
