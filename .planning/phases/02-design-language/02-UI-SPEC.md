## Spacing Scale

MUI's default `theme.spacing` unit is 8px. All spacing values must be expressed as multiples of `theme.spacing(N)`, which maps directly to the 8-point scale below.

| Token | Value | MUI spacing() call | Usage |
|-------|-------|--------------------|-------|
| xs | 4px | `theme.spacing(0.5)` | Icon gaps, inline padding within chips |
| sm | 8px | `theme.spacing(1)` | Compact element spacing, gap between paired buttons |
| md | 16px | `theme.spacing(2)` | Default element spacing, form field separation |
| lg | 24px | `theme.spacing(3)` | Section padding, vertical rhythm between content blocks |
| xl | 32px | `theme.spacing(4)` | Layout gaps between major sections |
| 2xl | 48px | `theme.spacing(6)` | Major section breaks |
| 3xl | 64px | `theme.spacing(8)` | Page-level spacing (e.g., top margin below AppBar) |

Exceptions:
- AppBar height: 64px (MUI Toolbar default) — not overridden
- Touch targets on mobile: minimum 44px height for all interactive elements (buttons, list items). MUI `size="medium"` buttons satisfy this; `size="small"` buttons must have `sx={{ minHeight: 44 }}` applied when rendered in touch contexts.

**Source:** CONTEXT.md — Claude's discretion (MUI default spacing confirmed as appropriate); touch target exception from DSNG-02.

## Typography

All heading roles use Josefin Sans. Body and label roles use Roboto (MUI default — do not override).

| Role | Size | Weight | Line Height | MUI variant | Usage |
|------|------|--------|-------------|-------------|-------|
| Display | 28px | 700 (bold) | 1.2 | h4 | Pod names, page-level headings |
| Heading | 20px | 700 (bold) | 1.2 | h6 | Section headings, tab content titles |
| Body | 16px | 400 (regular) | 1.5 | body1 | Primary readable content, list item text |
| Label | 14px | 400 (regular) | 1.4 | body2 | Secondary info, DataGrid cell text, chip labels |

Two weights only: 400 (regular) and 700 (bold). No intermediate weight.

Typography theme overrides in `createTheme`:
- `typography.fontFamily`: `'"Josefin Sans", "Roboto", sans-serif'` for display variants; body variants remain Roboto
- Override `h1`–`h4` and `h6` to use Josefin Sans explicitly via `typography.h4.fontFamily`, `typography.h6.fontFamily`, etc.
- `typography.h6.fontWeight`: `700`
- `typography.body1` and `typography.body2`: no font change (Roboto default)

**Source:** CONTEXT.md D-08, D-09 (fonts); sizes/weights are Claude's discretion — chosen to fit dark/gold MTG aesthetic and 3-4 size constraint. Weight reduced to 2 (400, 700) per design system constraint.

## Color

Dark theme throughout. All values are for MUI `palette` configuration in `createTheme({ palette: { mode: 'dark', ... } })`.

| Role | Value | MUI palette key | Usage |
|------|-------|-----------------|-------|
| Dominant (60%) | `#0f1117` | `palette.background.default` | App background, body fill |
| Secondary (30%) | `#1a1a2e` | `palette.background.paper` | Cards, elevated surfaces, AppBar |
| Accent (10%) | `#c9a227` | `palette.primary.main` | See accent reserved list below |
| Accent hover | `#e0b83a` | `palette.primary.light` | Hover state for accent elements |
| Primary text | `#e8edf5` | `palette.text.primary` | All primary readable text |
| Secondary text | MUI dark-mode default opacity applied to `#e8edf5` | `palette.text.secondary` | Secondary/supporting text |
| Destructive | MUI `error.main` (default `#f44336`) | `palette.error.main` | Destructive actions only |

**Accent reserved for:**
1. Primary `variant="contained"` buttons (e.g., "New Game", "Save Pod Name")
2. Active Tab indicator underline
3. Focused TextField outline
4. Chip `color="primary"` when used for role/status indicators that need emphasis

Accent is NOT used for: secondary buttons, general links, DataGrid header backgrounds, AppBar text, icon buttons by default.

**AppBar background:** `#1a1a2e` (same as `background.paper`) — coheres with surface palette, avoids contrast-break between AppBar and content. Override via `palette.AppBar` or `components.MuiAppBar` in theme. Source: CONTEXT.md D-07 (Claude's discretion — flat dark, cohesive with surface).

**gradientBackground CSS class:** Retire in favor of flat `#0f1117` background. The existing `.gradientBackground` class in `app/src/styles.css` should be removed or commented out; do not apply it anywhere in Phase 2 or later. Source: CONTEXT.md Specifics.

**Source:** CONTEXT.md D-01 through D-07; hover value and accent reserved list are Claude's discretion.

## Visual Hierarchy

**Primary focal point — Pod view:** Pod name rendered as Display/h4 (Josefin Sans, 28px, weight 700) at the top of the content area, below the AppBar. This is the dominant visual anchor on screen entry. The active tab's DataGrid or content block is the secondary focal point, receiving the user's attention after the pod name establishes context.

**Focal point sequence:**
1. Pod name (Display/h4) — establishes which pod the user is in
2. Tab row — shows available sections; active tab underlined in accent gold
3. Active tab content (DataGrid or Settings form) — primary work area

No decorative imagery or illustration is used; hierarchy is established entirely through typography scale and spacing.

## Component Overrides

The following `theme.components` overrides must be defined in `app/src/theme.ts` to enforce the design contract:

| Component | Override | Value | Rationale |
|-----------|----------|-------|-----------|
| MuiButton | `styleOverrides.root.borderRadius` | `6px` | Slightly rounded, not pill-shaped — fits clean/modern MTG aesthetic |
| MuiButton | `defaultProps.disableElevation` | `true` | Flat contained buttons; no shadow lift |
| MuiAppBar | `styleOverrides.root.backgroundColor` | `#1a1a2e` | Matches surface, overrides MUI primary-color AppBar default |
| MuiChip | `styleOverrides.root.fontFamily` | `'"Roboto", sans-serif'` | Keep chips in body font, not Josefin Sans |
| MuiTab | `styleOverrides.root.textTransform` | `'none'` | Tabs render in sentence case, not ALL CAPS (MUI default) |
| MuiDataGrid | (no override needed) | — | Dark mode adapts automatically when `palette.mode: 'dark'` is set |

**Source:** Claude's discretion — chosen to match dark/gold aesthetic and fix known MUI defaults that conflict with the design intent.

## Mobile Contract (DSNG-02)

Phase 2 must verify the Pod view is usable at 375px viewport width. The following constraints apply:

| Element | Requirement |
|---------|-------------|
| AppBar | Remains single-row at 375px; pod selector and avatar/logout collapse or are visible |
| Tabs | All 4 tabs (Decks, Players, Games, Settings) must be reachable — use `variant="scrollable" scrollButtons="auto"` on the Tabs component |
| DataGrid | `height: 400px` at mobile (reduce from 600px desktop); columns must not cause horizontal scroll — use `flex: 1` on primary column |
| Buttons | Minimum touch target 44px height; full-width `Button` acceptable on mobile if layout requires it |
| Settings form | `maxWidth: 500` container is acceptable; TextField at full width on mobile |
| Typography | Body text minimum 16px; never smaller than 14px for any readable content |

**Source:** DSNG-02 requirement; CONTEXT.md D-16.
