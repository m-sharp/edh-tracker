  <action>
In `spaHandler.ServeHTTP`, after the `os.Stat` check that determines the file does not exist on disk (line 40-42), add a check BEFORE the SPA fallback: if the requested path has a file extension (use `filepath.Ext(rel) != ""`), return a 404 instead of serving index.html. This distinguishes SPA route requests (no extension: `/pod/1`, `/player/2`) from static asset requests (has extension: `/static/js/main.abc123.js`, `/static/css/main.def456.css`).

Specifically, change the block at lines 40-42 from:
```go
if _, err := os.Stat(path); os.IsNotExist(err) {
    http.ServeFile(w, r, filepath.Join(absStatic, h.indexPath))
    return
}
```

To:
```go
if _, err := os.Stat(path); os.IsNotExist(err) {
    // If the path has a file extension, it is a static asset request (JS, CSS, image, etc.)
    // that is simply missing — return 404 instead of serving index.html.
    if filepath.Ext(rel) != "" {
        http.NotFound(w, r)
        return
    }
    http.ServeFile(w, r, filepath.Join(absStatic, h.indexPath))
    return
}
```

Also apply the same logic to the path-escape guard block (lines 33-37) — if the path escaped the static root AND has an extension, return 404 instead of index.html. Currently that guard unconditionally serves index.html which has the same problem.

UAT gap: "SPA handler in app/main.go falls back to index.html for ALL non-existent paths, including JS/CSS static asset requests."
