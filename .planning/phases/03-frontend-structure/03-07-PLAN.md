<objective>
Fix the CRA build configuration so that built asset paths are absolute (rooted at /) rather than relative (./). This resolves the blocker where refreshing any page at a sub-route (e.g. /pod/123) causes JS/CSS asset requests to resolve relative to that route (/pod/static/js/...) instead of from root (/static/js/...), which the SPA handler cannot find and falls back to index.html — producing "Uncaught SyntaxError: Unexpected token '<'" and a blank white screen.

Root cause: "homepage": "." in app/package.json causes CRA to emit ./static/... relative paths. At route /pod/123, ./static/js/main.js resolves to /pod/static/js/main.js — not found — SPA fallback serves index.html as JS.

The spaHandler in app/main.go is correct and does NOT need modification. The fix is entirely in app/package.json.

Purpose: UAT test #2 blocker -- app is unusable after any page refresh in production (Docker static server)
Output: Fixed app/package.json with homepage set to "/"
</objective>
