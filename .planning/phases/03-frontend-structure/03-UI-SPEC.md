## Spacing Scale

Inherited unchanged from Phase 2. MUI default `theme.spacing` unit = 8px.

| Token | Value | MUI spacing() call | Usage |
|-------|-------|--------------------|-------|
| xs | 4px | `theme.spacing(0.5)` | Icon gaps, inline padding within chips |
| sm | 8px | `theme.spacing(1)` | Compact element spacing, gap between paired buttons |
| md | 16px | `theme.spacing(2)` | Default element spacing, form field separation |
| lg | 24px | `theme.spacing(3)` | Section padding, vertical rhythm between content blocks |
| xl | 32px | `theme.spacing(4)` | Layout gaps between major sections |
| 2xl | 48px | `theme.spacing(6)` | Major section breaks |
| 3xl | 64px | `theme.spacing(8)` | Page-level spacing |

Exceptions:
- AppBar height: 64px (MUI Toolbar default — not overridden)
- Touch targets: minimum 44px height for all interactive elements on mobile. `size="medium"` buttons satisfy this. `size="small"` buttons in touch contexts must have `sx={{ minHeight: 44 }}`.

**Source:** Phase 2 UI-SPEC

## Typography

Inherited unchanged from Phase 2.

| Role | Size | Weight | Line Height | MUI variant | Usage |
|------|------|--------|-------------|-------------|-------|
| Display | 28px | 700 | 1.2 | h4 | Pod names, player names, deck names — page-level headings |
| Heading | 20px | 700 | 1.2 | h6 | Section headings within views, tab content titles |
| Body | 16px | 400 | 1.5 | body1 | Primary readable content, list item text |
| Label | 14px | 400 | 1.4 | body2 | Secondary info, DataGrid cell text, chip labels, timestamps |

Two weights only: 400 (regular) and 700 (bold). No intermediate weight.

**Source:** Phase 2 UI-SPEC + `app/src/theme.ts` (confirmed matching)

## Color

Inherited unchanged from Phase 2.

| Role | Value | MUI palette key | Usage |
|------|-------|-----------------|-------|
| Dominant (60%) | `#0f1117` | `palette.background.default` | App background, body fill |
| Secondary (30%) | `#1a1a2e` | `palette.background.paper` | Cards, elevated surfaces, AppBar |
| Accent (10%) | `#c9a227` | `palette.primary.main` | See accent reserved list below |
| Accent hover | `#e0b83a` | `palette.primary.light` | Hover state for accent elements |
| Primary text | `#e8edf5` | `palette.text.primary` | All primary readable text |
| Secondary text | MUI dark-mode default opacity on `#e8edf5` | `palette.text.secondary` | Secondary/supporting text |
| Destructive | MUI `error.main` (`#f44336`) | `palette.error.main` | Destructive actions only |

**Accent reserved for:**
1. Primary `variant="contained"` buttons
2. Active Tab indicator underline
3. Focused TextField outline
4. `Chip color="primary"` for role/status indicators that need emphasis

Accent is NOT used for: secondary buttons, general links, DataGrid header backgrounds, AppBar text, icon buttons by default.

**Source:** Phase 2 UI-SPEC + `app/src/theme.ts` (confirmed matching)

## Per-View Audit (DSNG-04)

Per CONTEXT.md D-20 through D-25: audit scope is 7 views at 375px (mobile-first). Issues only — no "things working well" items. Each issue: current behavior → desired behavior.

---

### View 1: LoginView (`app/src/routes/login.tsx`)

**Issue L-01 — typography: heading not using Display role**
Current: `<Typography variant="h4">EDH Tracker</Typography>` — renders correctly as Josefin Sans 28px.
Desired: No change needed — this is correct. (Non-issue, confirmed.)

**Issue L-02 — spacing: top margin lacks vertical centering on tall viewports**
Current: `mt: 8` (64px top margin) — on a 375×667px phone viewport this sits at the top third, not vertically centered.
Desired: Replace `mt: 8` with flexbox vertical centering: parent `Box` uses `minHeight: "100vh"` with `justifyContent: "center"` in addition to `alignItems: "center"`. This centers the login block on all viewport heights.

**Issue L-03 — interaction: no loading state after button click**
Current: The "Sign in with Google" button triggers an `href` redirect. No feedback between click and page navigation.
Desired: No change needed — href redirects are instant browser navigations; a loading state would flash and disappear. Acceptable as-is.

**Issue L-04 — spacing: no app logo or visual identity beyond the text heading**
Current: Just text + button — no playing cards icon from the AppBar.
Desired: Add the `SvgIconPlayingCards` icon above the "EDH Tracker" heading at a larger size (`fontSize: 48px`) so the login page has consistent visual identity with the rest of the app. This is the one place users see the brand without the AppBar.

---

### View 2: HomeView (`app/src/index.tsx` → moves to `app/src/routes/home/index.tsx`)

**Issue H-01 — interaction: loading flash**
Current: `<Typography>No pods yet...</Typography>` renders immediately before `GetPodsForPlayer` resolves. Users see the empty state message on every page load, even when they have pods.
Desired: While `GetPodsForPlayer` is pending, render nothing (or a centered `<CircularProgress>`). Only render the "No pods yet" message after the fetch resolves and `pods.length === 0` is confirmed. Pattern: add `loading` state initialized to `true`, set to `false` in `.then()` and `.catch()`.

**Issue H-02 — layout: no container/padding for the empty state message**
Current: `<Typography>` renders flush at the top of the content area with no padding or centering.
Desired: Wrap in a `Box` with `display: "flex", flexDirection: "column", alignItems: "center", pt: 4` so the message is centered and has breathing room.

**Issue H-03 — copy: empty state message is incomplete**
Current: "No pods yet. Create your first pod to get started."
Desired: "No pods yet. Create your first pod or ask a manager for an invite link." (aligned with Phase 2 copywriting contract)

---

### View 3: PodView (`app/src/routes/pod.tsx` → splits into `pod/` subdirectory)

**Issue P-01 — spacing: DataGrid height fixed at 600px on all viewports**
Current: `height: 600` on both `PodDecksTab` and `PodGamesTab` DataGrid wrappers.
Desired: Use responsive height: `height: { xs: 400, sm: 600 }` so the grid does not over-extend at 375px. (Phase 2 mobile contract specified 400px at mobile.)

**Issue P-02 — interaction: Settings tab Settings tab visibility bug at index 3**
Current: `{tab === 3 && isManager && <PodSettingsTab />}` — if a non-manager is on index 3, they see nothing (no error). After TabbedLayout migration this is resolved structurally via `hidden` prop.
Desired: After TabbedLayout migration, use `hidden: !isManager` on the Settings tab config. Issue closes with FEND-02 implementation.

**Issue P-03 — typography: Pod name heading has `sx={{ mb: 2 }}` spacing**
Current: `mb: 2` (16px) below pod name before tabs.
Desired: Retain `mb: 2`. This is correct per spacing scale. (Non-issue, confirmed.)

**Issue P-04 — interaction: Players tab — role chip label casing inconsistent**
Current: `label={p.role === "manager" ? "Manager" : "Member"}` — already title-cased.
Desired: No change needed. (Non-issue, confirmed.)

**Issue P-05 — interaction: Players tab — Promote/Remove buttons are `size="small"` without 44px min-height**
Current: `<Button size="small">` for Promote and Remove actions — touch target is below 44px minimum.
Desired: Add `sx={{ minHeight: 44 }}` to both Promote and Remove buttons in `PodPlayersTab`. This applies the Phase 2 mobile touch-target exception.

**Issue P-06 — layout: Settings tab — TextField + Button row overflows on narrow viewport**
Current: `Box sx={{ display: "flex", gap: 1 }}` containing a TextField and "Save Pod Name" Button — on 375px these crowd and may clip.
Desired: Wrap the row in `flexWrap: "wrap"` and set the TextField to `fullWidth` when the row wraps: `sx={{ display: "flex", gap: 1, flexWrap: "wrap" }}` with `TextField sx={{ flex: 1, minWidth: 160 }}`. Same pattern for the invite link row.

**Issue P-07 — interaction: AppBar title clips on mobile when pod selector is rendered**
Current: "EDH Tracker" text + PodSelector + Avatar + Logout all on one row — at 375px these compete for space and the title clips or overlaps. (Known deferred issue from Phase 2, DSNG-02.)
Desired: On `xs` breakpoint, hide the "EDH Tracker" text label: `sx={{ display: { xs: "none", sm: "flex" } }}`. The playing cards icon remains visible as brand anchor. This is the minimal fix; a mobile menu/drawer is Phase 5+.

### View 4: PlayerView (`app/src/routes/player.tsx` → splits into `player/` subdirectory)

**Issue PL-01 — typography: stats row uses raw `<span>` with `<strong>` instead of MUI Typography**
Current: `<span><strong>Games Played:</strong> {player.stats.games}</span>` — styling is browser-default, not MUI token-based.
Desired: Replace with `<Typography variant="body1"><strong>Games Played:</strong> {player.stats.games}</Typography>` or a `Box` + two `Typography` children. Use `variant="body2"` for the label, `variant="body1"` for the value, consistent with the type scale.

**Issue PL-02 — layout: stats row uses `justify-content: space-evenly` with no padding on mobile**
Current: `Box sx={{ display: "flex", flexDirection: "row", justifyContent: "space-evenly", py: 1 }}` — three stat items on one row. At 375px the items touch the viewport edges.
Desired: Add `px: 1` (8px horizontal padding) and allow wrapping: `flexWrap: "wrap", gap: 2` so stats wrap to a second row if needed rather than clipping.

**Issue PL-03 — interaction: Player tabs missing `variant="scrollable" scrollButtons="auto"`**
Current: `<Tabs value={tab} onChange={...} sx={{ mb: 2, mt: 2 }}>` — no scroll variant.
Desired: Add `variant="scrollable" scrollButtons="auto"` to match the Pod view and the upcoming TabbedLayout default. Resolves with FEND-02 TabbedLayout migration.

**Issue PL-04 — typography: "Player created at:" metadata uses `<em>` instead of MUI Typography**
Current: `<em>Player created at: {new Date(player.created_at).toLocaleString()}</em>` — italicized browser-default text.
Desired: `<Typography variant="body2" color="text.secondary">Created: {new Date(player.created_at).toLocaleString()}</Typography>` — aligns with the Label role and secondary text color from the design system. Remove `<em>` wrapper.

**Issue PL-05 — typography: Error loading pods uses `<Typography color="error">Error loading pods: {error.message}</Typography>`**
Current: Error message exposes `.message` directly to the user.
Desired: Show a fixed string: `<Typography variant="body2" color="error">Could not load pods. Refresh to try again.</Typography>` — no raw error message exposure.

**Issue PL-06 — layout: Player Settings — Create New Pod section is misplaced**
Current: "Create New Pod" lives in PlayerSettingsTab. CONTEXT.md notes this is a known issue (TODO comment in source).
Desired: Move pod creation to a dedicated location in Phase 5 (POD-01). In Phase 3, the section stays in Settings but gets clear visual separation: add a `Divider` above the "Create New Pod" section with a note-style `Typography variant="body2" color="text.secondary"` subtitle: "This will move to the pod page in a future update." This is a bridge solution only.

**Issue PL-07 — interaction: Player Settings Leave Pod buttons are `size="small"` without 44px min-height**
Current: `<Button size="small" color="error">Leave</Button>` — touch target below 44px minimum.
Desired: Add `sx={{ minHeight: 44 }}` to the Leave button. Same fix as P-05.

**Issue PL-08 — error: PlayerDecksTab and PlayerGamesTab error state uses raw `<span>` element**
Current: `return <span>Error Loading Player's Decks: {error.message}</span>`
Desired: `return <Typography variant="body2" color="error">Could not load decks. Refresh to try again.</Typography>` — consistent error pattern, wrapped in a `Box` with padding.

Current: `Box sx={{ display: "flex", flexDirection: "row", gap: 4 }}` — at 375px three items at 32px gap may not fit without overflow.
Desired: Add `flexWrap: "wrap"` and `gap: 2` (16px) to the stats row. Same fix as PL-02.

**Issue D-04 — interaction: DeckView tabs missing `variant="scrollable" scrollButtons="auto"`**
Current: `<Tabs value={tab}>` — no scroll variant.
Desired: Add `variant="scrollable" scrollButtons="auto"`. Resolves with FEND-02 TabbedLayout migration.

### View 6: GameView (`app/src/routes/game.tsx` → moves to `game/index.tsx`)

**Issue G-01 — typography: game heading uses raw `<h1>` instead of MUI Typography**
Current: `<h1>{pod.name} — Game #{game.id}</h1>` — browser-default heading style, not Josefin Sans, not theme-controlled.
Desired: `<Typography variant="h4">{pod.name} — Game #{game.id}</Typography>` — applies the Display role (Josefin Sans, 28px, weight 700) consistently with other view headings.

**Issue G-02 — typography: game date uses `<em>` instead of MUI Typography**
Current: `<em>{new Date(game.created_at).toLocaleString()}</em>` — browser-default italic.
Desired: `<Typography variant="body2" color="text.secondary">{new Date(game.created_at).toLocaleString()}</Typography>` — Label role, secondary text color.

**Issue G-03 — interaction: EditIcon button in GameDescription has no tooltip**
Current: `<IconButton size="small" onClick={() => setEditing(true)}><EditIcon /></IconButton>` — TODO comment confirms this is known.
Desired: Wrap in `TooltipIconButton` (new shared component from FEND-03): `<TooltipIconButton title="Edit description" icon={<EditIcon fontSize="small" />} onClick={() => setEditing(true)} />`.

**Issue G-04 — interaction: EditIcon and DeleteIcon buttons in GameResultsGrid have no tooltips**
Current: `<IconButton size="small"><EditIcon /></IconButton>` and `<IconButton size="small" color="error"><DeleteIcon /></IconButton>` — no tooltip context.
Desired: Wrap both in `TooltipIconButton`: "Edit result" and "Remove result" respectively.

**Issue G-05 — layout: GameView content uses `alignItems: "center"` which over-centers the DataGrid**
Current: `<Box sx={{ display: "flex", flexDirection: "column", alignItems: "center" }}>` — the DataGrid stretches to full width of its container but the containing Box aligns center, which can cause layout narrowing on some viewports.
Desired: Change `alignItems: "center"` to `alignItems: "stretch"` for the outer column Box. The `GameDescription` Stack can individually remain left-aligned. Keep the heading centered if desired, or left-align for consistency with other views. Recommendation: left-align to match Pod/Player/Deck heading treatment.

**Issue G-06 — layout: DataGrid height fixed at 355px with no responsive handling**
Current: `<Box sx={{ height: 355, width: "100%" }}>` — at 375px this may cut off rows if many results.
Desired: Allow height to grow: `sx={{ height: "auto", minHeight: 200, width: "100%" }}` with `autoHeight` prop on `<DataGrid>`. For a results grid with typically 4–6 rows, `autoHeight` is appropriate and avoids wasted space. Alternatively, `height: { xs: 300, sm: 400 }` if autoHeight has performance concerns.

**Issue G-07 — layout: DeleteGameButton has `sx={{ mt: 2 }}` — needs consistent bottom margin**
Current: `<Button ... sx={{ mt: 2 }}>Delete Game</Button>` — bottom of view has no bottom padding.
Desired: The parent view's Container (from `root.tsx`) already applies `mb: 5` at the page level. No additional bottom margin needed. Remove the `mt: 2` and add `mt: 3` for breathing room above the delete button. (Non-issue with `root.tsx` container margin — retain as-is or adjust to `mt: 3`.)
